# 技术实现文档

## 架构概述

本项目采用组件化架构，将游戏逻辑拆分为独立的实体、系统和工具模块，便于维护和扩展。

### 核心设计原则

1. **关注点分离**：渲染、逻辑、输入、音频各自独立
2. **对象池**：子弹、敌人、粒子使用对象池复用，避免频繁 GC
3. **固定时间步**：游戏逻辑以 60 FPS 的固定步长更新，保证物理一致性
4. **状态机**：菜单/游戏中/暂停/结束四个状态清晰切换

## 关键算法

### 1. 游戏循环 (Game Loop)

使用固定时间步长更新 + 可变帧率渲染：

```
accumulator += deltaTime
while (accumulator >= FIXED_DT) {
    update(FIXED_DT)
    accumulator -= FIXED_DT
}
render()
```

- 防止帧率波动导致物理计算不一致
- 在低速设备上自动降帧，但逻辑仍正确

### 2. 碰撞检测

采用轴对齐距离检测（圆形碰撞体）：

```
distance(a, b) < a.radius + b.radius
```

所有实体使用圆形碰撞体，计算开销低且足够精确。

### 3. 敌人 AI

三种敌人共享同一套寻路逻辑，仅参数不同：

- **向量计算**：`dir = normalize(player.position - enemy.position)`
- **快速型**：在前进方向基础上叠加正弦波横向偏移，形成蛇形走位
- **坦克型**：直线追击，高血量
- **追击型**：标准直线追击

### 4. 对象池 (ObjectPool)

预创建 + 复用模式：

```typescript
class ObjectPool<T> {
    private available: T[] = []
    acquire(): T { /* 复用或新建 */ }
    release(item: T) { /* 重置状态并归还 */ }
}
```

对象池用于子弹（50）、敌人（50）、粒子（100），在高频场景下显著减少内存分配。

### 5. 粒子系统

每个粒子维护位置、速度、生命周期、颜色：

- **爆炸**：从中心向四周放射，速度随机，带重力衰减
- **尾焰**：子弹后方生成，速度略低于子弹
- **受击**：命中点生成少量粒子

渲染时使用 `globalAlpha` 根据生命周期做淡出效果。

### 6. 波次系统 (Wave System)

```
totalEnemies = baseEnemies + wave * enemyIncrement
spawnInterval = max(minInterval, baseInterval - wave * 100)
```

- 每波敌人数线性增长
- 生成间隔随波次递减（有下限）
- 击杀全部敌人后进入波次间隔（3秒），然后自动进入下一波

### 7. 音频生成

使用 Web Audio API 实时合成音效，无需外部音频文件：

- **射击**：方波频率从 800Hz 快速衰减到 200Hz
- **爆炸**：白噪声 + 低通滤波器衰减
- **拾取**：正弦波频率上升
- **游戏结束**：四个递减音符序列

## 性能优化

1. **对象池**：减少运行时对象分配
2. **批量渲染**：Canvas 状态切换最小化
3. **空间分割简化**：屏幕内实体数量可控，无需四叉树
4. **粒子数量控制**：单帧上限通过对象池自然限制
5. **响应式缩放**：Canvas 实际分辨率固定 1280x720，CSS 缩放适配屏幕

## 扩展建议

- 添加更多敌人类型：实现新的 `Enemy` 子类或扩展 `typeName` 分支
- 添加 Boss 战：在特定波次生成大型敌人，增加血条和阶段切换
- 本地排行榜：使用 `localStorage` 存储最高分
- 联网对战：引入 WebSocket 实现双人对战或排行榜同步
